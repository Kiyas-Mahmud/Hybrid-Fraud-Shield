4. THE 11 BASE MODELS
-----------------------------------------------------------------------------

ML MODELS (Traditional Machine Learning):
1. Logistic Regression
   - Fast, simple, interpretable
   - Good for linear patterns
   
2. Random Forest
   - Handles non-linear patterns
   - Good with imbalanced data
   
3. XGBoost
   - Powerful gradient boosting
   - Best ML performer (usually)
   
4. CatBoost
   - Handles categorical features well
   - Robust to overfitting
   
5. LightGBM
   - Fast training, good accuracy
   - Efficient for large datasets

DL MODELS (Deep Learning Neural Networks):
6. FNN (Feedforward Neural Network)
   - General-purpose deep learning
   - Best DL model in our tests (F1: 0.51)
   
7. CNN (Convolutional Neural Network)
   - Detects local patterns
   - Spatial feature learning
   
8. LSTM (Long Short-Term Memory)
   - Temporal pattern detection
   - Sequential dependencies
   
9. BiLSTM (Bidirectional LSTM)
   - Reads patterns forward & backward
   - Richer context understanding
   
10. Hybrid (CNN-BiLSTM)
    - Combines spatial + temporal
    - Complex pattern detection
    
11. Autoencoder
    - Anomaly detection
    - Reconstruction error approach

META-LEARNER (The Chief Judge):
â€¢ Algorithm: Logistic Regression (C=100, penalty=l2)
â€¢ Input: 11 predictions from base models
â€¢ Output: Final fraud probability
â€¢ Trained to know which models are most reliable
â€¢ Optimal threshold: 0.89 (89%)

7. TRAINING PROCESS
-----------------------------------------------------------------------------

Dataset: 
â€¢ 590,540 transactions total
â€¢ 71 features per transaction
â€¢ 4.18% fraud rate (highly imbalanced)
â€¢ Split: 60% train, 20% validation, 20% test

Step 1: Train 5 ML models independently
        Time: ~2 hours
        Best: XGBoost (F1: 0.48)

Step 2: Train 6 DL models independently
        Time: ~8 hours (with hyperparameter tuning)
        Best: FNN (F1: 0.51)

Step 3: Generate predictions on validation set
        All 11 models predict on 118,108 samples

Step 4: Train meta-learner on base predictions
        Input: 11 predictions per sample
        Output: Final fraud probability
        Time: ~5 minutes

Step 5: Optimize decision threshold
        Tested thresholds: 0.1 to 0.99
        Optimal: 0.89 (maximizes F1-score)

Step 6: SHAP analysis for explainability
        Compute SHAP values for all predictions
        Generate visualizations and reports

Total Training Time: ~12 hours on GPU
